## 原码、反码、补码

### 原码

原始二进制，左边是高位，右边是低位，最高位是符号位，**0为 +，1为 -**。

```
+1 -> 0000 0001
+0 -> 0000 0000
-0 -> 1000 0000
-1 -> 1000 0001
```

原码存储导致2个问题：
1. 出现两种0的存储方式。
2. 正数和负数相加，结果不正确，计算机只会计算加法。

```
计算 1 - 1
+1 0000 0001
-1 1000 0001
   1000 0010 = -2
```

### 反码

1. 正数的原码和反码保持一致。
2. 只求负数的反码。
3. 在原码基础上，符号位不变，其他位取反。

```
+1 -> 0000 0001
+0 -> 0000 0000
-0 -> 1111 1111
-1 -> 1111 1110
```

反码存储导致1个问题：

1. 出现两种0的存储方式。

```
计算 1 - 1
+1 0000 0001
-1 1111 1110
   1111 1111 = -0
```

### 补码

为解决以上问题，计算机采用补码存储。

1. 对于正数的原码、反码、补码都保持一致。
2. 对于负数，补码为其反码+1。

```
+1 ->  0000 0001
+0 ->  0000 0000
-0 -> 10000 0000 (最高位丢弃)
-1 ->  1111 1111
```

```
计算 1 - 1
+1  0000 0001
-1  1111 1111
   10000 0000 = 0
```
